Librarie client de l'API CyberMailing.

Introduction

Les instructions qui suivent correspondent à l'installation de la libraire seule.
Si cette librairie est incluse dans un plugin ou un module, ignorez ces instructions
et reportez vous aux instructions du plugin.
Cette libraire permet aux sites des utilisateurs de cyberMailing de communiquer avec la base de donnée
de CyberMailing.

Elle est composée de 2 fonctions primaires :

1. la fonction "écoute" (listen) qui permet à votre site de recevoir des informations
en provenance de cybermailing. CyberMailing y communique les informations
liées à l'inscription et à la désinscription des abonnés.

2. La fonction "parole" (talk) qui permet à votre site de transmettre des informations
à cybermailing. Cette partie vous permet de réaliser des opérations sur les abonnés
directement de serveur à serveur et d'utiliser les fonctions de désinscription
de confirmation et de tracking directement sur votre site.

Pré-Requis

Apahe2, PHP 5.3+
(Mysql 5.+  en option pour la partie écoute, si l'option sauvegarde sous mysql est utilisée).

Installation

Remarque :
Les 2 fonctions primaires sont indépendantes.
Si vous souhaitez n'utiliser qu'une fonction, vous pouvez ignorer tout ce qui concerne l'autre.

1. Dézipper l'archive sur votre ordinateur

Vous obtenez un dossier /cybermailing-api-client

2. Modifier les fichiers de configuration

A. Fonction Parole /cybermailing-api-client/config.php


	Renomez le fichier config.php en config-me.php, ainsi vos configurations ne seront pas changées par d'éventuelles nouvelles mises à jour.
		
	Sur la première ligne, indiquez votre clef secrête que vous trouvez dans votre zone profil.
	Voir votre interface cybermailing : "Mon Compte" >> "Mon Profil".

	En option, sur la deuxieme ligne, vous pouvez aussi y modifier le nom de la variable de tracking
	si celle par défaut ne vous convient pas.

B. Fonction Ecoute /cybermailing-api-client/listen/definition.php

Modifiez le fichier	: 

	Renomez le fichier definition.php en definition-me.php, ainsi vos configurations ne seront pas changées par d'éventuelles nouvelles mises à jour.

	Il contient des explications, il s'agit des informations nécessaires pour utiliser
	les fonctionalités de sauvegarde disponibles (accès mysql, adresses emails, suffix
	du nom du fichier texte de sauvegarde).

	Si vous ne souhaitez pas utiliser une fonctionalité, vous pouvez laisser les valeurs par défaut.

	Pour préciser vos fonctionalités et les utiliser continuez ici

3. Activez cette librairie client

Ajoutez les 2 lignes suivantes au .htaccess de votre site :
	

	AddType application/x-httpd-php .html .htm 
	php_value auto_prepend_file "/home/web/votresite/cybermailing-api-client/_init.php"
	


	Remarque PRESTASHOP :
	Le fichier .htacess pouvant être modifié par prestashop, ces lignes doivent être insérées
	via le générateur de .htacess qui se trouve dans le backOffice.
	Via : "outils" >> "generateurs" >> "configuration specifique"
	
	Vous devrez toutefois remplacer /home/web/votresite/ par le chemin absolu de votre site sur le serveur
	sur lequel il est installé. Si vous ne connaissez pas cette information consultez le guide détaillé pour
	des instructions vous permettant d'obtenir cette information.

	Si votre site n'a pas de fichier .htaccess, créez un fichier avec un éditeur de texte simple
	(type bloc note / notepad, surtout pas word !) que vous nommez .htaccess
	(il n'y a rien avant le point) et placez y la ligne ci-dessus en ayant modifié /home/web/votresite/
	par votre chemin
Pour exploiter les possibilités de la fonction parole consultez :
	Le guide rapide
	ou Le guide détaille

4. Chargez le dossier /cybermailing-api-client à la racine de votre site.

Félicitation vous avez terminé !
